GreenThread is a GPU inference platform for Kubernetes that lets you serve dozens of large language models on shared GPUs by intelligently swapping them in and out of GPU memory. Instead of dedicating GPUs to individual models, GreenThread keeps sleeping models checkpoint on NVMe or in CPU memory and wakes them on demand — in under 2 seconds.

## How it works

GreenThread deploys models as Kubernetes custom resources. Each model gets its own pod with a sidecar proxy that manages the sleep/wake lifecycle. When a request arrives:

1. If the model is **serving**, the request is proxied directly to vLLM — no overhead.
2. If the model is **sleeping**, the sidecar wakes it by restoring weights from CPU cache to GPU, then proxies the request.
3. If GPUs are full, the [fairness policy](/fairness-policy) preempts the least-recently-used model to make room.

All of this happens transparently behind OpenAI-compatible, Anthropic-compatible, and other standard inference APIs.

Each model runs in its own pod with a sidecar that handles request queuing, wake-on-request, and sleep lifecycle. When a model sleeps, it performs a CUDA checkpoint that releases **all** GPU memory — true zero VRAM — making the GPU fully available for other models.

## Performance

For a 20B-parameter model:

| Operation | Time |
|-----------|------|
| Traditional cold start | ~40 seconds |
| GreenThread wake (from CPU RAM) | ~600ms |
| GreenThread wake (from NVMe with GDS) | ~2 seconds |
| GreenThread sleep | ~90 milliseconds |

The improvement comes from keeping the vLLM process alive while sleeping and using fast transfer paths — DMA from pinned host memory at ~65 GB/s, or GPU Direct Storage from NVMe at ~20 GB/s.

## API compatibility

GreenThread exposes inference endpoints through a per-model gateway path. Each model supports the full vLLM API surface:

- `POST /<model>/v1/chat/completions` — Chat completions with streaming
- `POST /<model>/v1/completions` — Text completions
- `POST /<model>/v1/responses` — OpenAI Responses API
- `POST /<model>/v1/messages` — Anthropic Messages API
- `POST /<model>/v1/embeddings` — Text embeddings
- `GET /<model>/v1/models` — List loaded models

See [API Reference](/inference-api) for the full endpoint list.

## Quickstart

Deploy a model:

```yaml
apiVersion: greenthread.ai/v1alpha1
kind: Model
metadata:
  name: llama-3-8b
  namespace: greenthread-system
spec:
  modelName: meta-llama/Llama-3.1-8B-Instruct
  dtype: bfloat16
  gpuMemoryUtilization: "0.90"
  tensorParallelSize: 1
  replicas: 1
  extraArgs:
    - "--enforce-eager"
```

```bash
kubectl apply -f model.yaml
```

Send a request through the gateway:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://<gateway-url>/llama-3-8b/v1",
    api_key="not-needed",
)

response = client.chat.completions.create(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(response.choices[0].message.content)
```

If the model is sleeping, GreenThread automatically wakes it, serves the request, and returns the response — all behind the standard API.
