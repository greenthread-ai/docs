GreenThread is a GPU inference platform that lets you serve dozens of large language models on shared GPUs by intelligently swapping them in and out of GPU memory. Instead of dedicating GPUs to individual models, GreenThread keeps sleeping models on fast storage and wakes them on demand — in under a second.

## How it works

When a request arrives for a model:

1. If the model is **serving**, the request is forwarded directly — no overhead.
2. If the model is **sleeping**, GreenThread wakes it by restoring weights to GPU, then forwards the request.
3. If GPUs are full, the [fairness policy](/fairness-policy) preempts the least-recently-used model to make room.

All of this happens transparently behind OpenAI-compatible, Anthropic-compatible, and other standard inference APIs.

When a model sleeps, GreenThread releases **all** GPU memory — true zero VRAM — making the GPU fully available for other models. No dedicated GPU per model, no idle waste.

## Performance

For a 20B-parameter model:

| Operation | Time |
|-----------|------|
| Traditional cold start | ~40 seconds |
| GreenThread wake (from RAM) | ~600ms |
| GreenThread wake (from NVMe) | ~2 seconds |
| GreenThread sleep | ~90 milliseconds |

## API compatibility

GreenThread exposes standard inference APIs per model. Any SDK or tool that works with OpenAI or Anthropic works with GreenThread:

- `POST /<model>/v1/chat/completions` — Chat completions with streaming
- `POST /<model>/v1/completions` — Text completions
- `POST /<model>/v1/responses` — OpenAI Responses API
- `POST /<model>/v1/messages` — Anthropic Messages API
- `POST /<model>/v1/embeddings` — Text embeddings
- `GET /<model>/v1/models` — List loaded models

See [API Reference](/inference-api) for the full endpoint list.

## Quickstart

Point your OpenAI SDK at your GreenThread instance:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://<greenthread-url>/llama-3-8b/v1",
    api_key="not-needed",
)

response = client.chat.completions.create(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(response.choices[0].message.content)
```

If the model is sleeping, GreenThread automatically wakes it, serves the request, and returns the response — all behind the standard API.
